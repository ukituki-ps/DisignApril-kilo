# Design System Roadmap

Приоритеты развития дизайн-системы на ближайшие итерации.

## P0 - Стабильность и контракт

- Зафиксировать стабильный публичный API `@april/ui` и `@april/tokens`.
- Свести к минимуму неявные breaking changes в экспортах.
- Ввести единый шаблон задач и отчетов для инженерных изменений.

## P1 - Качество компонентов

- Расширить showcase сценариями проверки состояний и edge-cases.
- Стандартизировать паттерны для форм, таблиц, карточек и навигации.
- Снизить дублирование стилей через утилиты и тему.

## P2 - DX и масштабирование

- Упростить интеграцию DS в микрофронты через понятные гайды.
- Формализовать процесс релизов и changelog для пакетов.
- Добавить практики регрессионной проверки визуального поведения.

## Метрики прогресса

- Доля компонентов, покрытых showcase-сценариями.
- Количество правок, требующих ручного хотфикса у потребителей.
- Время от постановки задачи до готового и документированного изменения.
