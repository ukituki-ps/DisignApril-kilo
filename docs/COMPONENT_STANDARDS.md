# Component Standards

Единые требования к компонентам в `@april/ui`.

## API и структура

- Компонент имеет предсказуемый и минимальный API (без лишних пропсов).
- Значимые пропсы типизированы и описаны понятными именами.
- Экспорт добавляется в `packages/ui/src/index.ts` только после проверки необходимости публичного API.

## Визуальные правила

- Компонент корректно работает в light/dark теме.
- Компонент поддерживает режимы плотности (comfortable/compact), если влияет на размеры.
- Цвета и отступы строятся на токенах и теме, а не на случайных hex.

## UX и a11y

- Есть текстовая метка/aria-label для интерактивных элементов.
- Иконки: декоративные — через `AprilIcon` рядом с видимым текстом или с `aria-hidden`; **только иконка** у `Button` / `ActionIcon` — обязательный доступный имя (`aria-label`, текст в `Tooltip` с фокусируемым контролом и т.д.). См. витрину **§19. Иконки** в `UIKit`.
- Сегментированный переключатель, где в сегменте **только иконка** — у каждого сегмента должно быть доступное имя (например `VisuallyHidden`); см. **§20** в `UIKit` и [доку Mantine — SegmentedControl](https://mantine.dev/core/segmented-control/#accessibility-and-usability).
- Модальные формы и подтверждения: предпочтительно **`AprilModal`** — основные и вторичные действия в **шапке справа** (рядом с закрытием), **скролл только у тела**; не перекрывайте поля «плавающими» кнопками без визуального разделения шапки и контента. См. **§12** в `UIKit` и `DESIGN_SYSTEM.md` §11.
- **Mobile** (&lt;768px): фиксированные действия текущего слоя — в **`AprilMobileShellBar`**; в панели **один активный контекст за раз** — либо shell экрана (табы, действия, поиск), либо **только** действия верхнего модального слоя; не смешивать табы приложения с кнопками «Отмена / Сохранить» открытой модалки. См. `DESIGN_SYSTEM.md` §8 Mobile и §11 «Паттерн: формы и подтверждения».
- Состояния `hover`, `focus`, `disabled`, `error` визуально различимы.
- Ошибки формы показываются рядом с контролом и имеют понятный текст.

## Инженерные критерии

- Нет дублирования логики, которую можно вынести в общий utility/helper.
- Компонент не тянет тяжелые зависимости без необходимости.
- Showcase содержит пример базового использования и минимум один edge-case.
- JSON: см. секцию **18. JSON & JSON Schema (дерево + форма)** в `UIKit` (`AprilJsonTreeEditor`, `AprilJsonSchemaForm`, валидация, read-only).
- Иконки: см. секцию **19. Иконки** в `UIKit` (`AprilIcon`, курируемые `AprilIcon*`).
- Сегменты `SegmentedControl` **только с иконками** — скрытое текстовое имя (`VisuallyHidden` и т.п.), см. **§20. Градиентный SegmentedControl** в `UIKit` (`AprilGradientSegmentedControl`).

## Checklist перед merge

- [ ] Проверен компактный режим (если применимо).
- [ ] Компонент отображается корректно в showcase.
- [ ] Обновлена документация, если изменились правила или API.
