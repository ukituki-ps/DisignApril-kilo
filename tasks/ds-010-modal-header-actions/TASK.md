## Цель
Закрепить в дизайн-системе April **нормативный паттерн модальных окон**: основные и вторичные действия (подтверждение, отмена и т.п.) **всегда в шапке справа** (в одной зоне с кнопкой закрытия), с **неподвижной шапкой** при вертикальном скролле контента. Привести **витрину и компоненты пакета `@ukituki-ps/april-ui`** к этому паттерну, чтобы продукты не дублировали разнобой «кнопки снизу vs сверху».

**Зачем:** предсказуемая зона действий при переключении между формами, меньше движения указателя, отсутствие «плавающего» низа у форм разной высоты.

## Контекст
- Репозиторий: `DisignApril`, пакет `packages/ui`, peer `@mantine/core` ^7.
- Сейчас кнопки в примерах чаще в **низу** тела модалки (`Stack` + `Group justify="flex-end"`): см. `packages/ui/src/components/ModalSection.tsx`, `SafetyPatterns.tsx` (модалка удаления), тесты/смоки, которые ориентируются на эти разметки.
- `CardListColumn` встраивает несколько `Modal` с `title` строкой и телом из `renderFilterModal` / `renderSortModal` / fallback add — поведение шапки и действий **зависит от содержимого колбэков**; в `CardListColumnSection` и тестах есть эталонные разметки.
- Иконки UI в DS: `AprilIcon*` / `lucide-react` (см. `DESIGN_SYSTEM.md` §11) — для кнопок в шапке при необходимости использовать согласованный набор и **обязательные имена для a11y** (`aria-label`, tooltip при иконках без текста).

### 2. Техническая реализация в `packages/ui`
Выбрать и задокументировать в PR **один** устойчивый подход (не смешивать без причины):

- Шапка **визуально и при скролле** остаётся на месте; контент формы прокручивается.
