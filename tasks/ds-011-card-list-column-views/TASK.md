- ID / ветка: `DS-011` / `feature/ds-011-card-list-column-views`
- Приоритет: medium

## Цель
Расширить **`CardListColumn`** в `@ukituki-ps/april-ui` тремя явными **видами отображения** (свернутая колонка / список / сетка карточек), переключаемыми **одной кнопкой по кругу**, с улучшенным контентом в каждом виде. Сохранить для продуктов **один публичный импорт** компонента (фасад: внутренняя нарезка по файлам допустима и желательна).

**Зачем:** плотный контекст при свёрнутой колонке (выбранная сущность), быстрый скан списка с визуальным якорем (аватар/иконка), альтернативный обзор «доска» на доступную область без отдельного экрана в продукте.

## Контекст
- Репозиторий: `DisignApril`; пакет: `packages/ui`; витрина: `apps/showcase`.
- Текущая реализация: `packages/ui/src/components/CardListColumn.tsx` — бинарное `collapsed`, режимы `inline` | `overlay`, ресайз ширины, `CardListColumnItem` с `id`, `title`, `description`, …; **выбранного элемента и URL картинки в типе нет**.
- Постановка продолжает линию `tasks/ds-001-card-list-column` и не отменяет соглашения по модалкам (`AprilModal`, DS-010).
- Внешний референс плотности карточек в сетке: [Tasks card | Mantine UI](https://ui.mantine.dev/component/task-card/) — ориентир по иерархии и плотности; цвета и типографика — по April (`DESIGN_SYSTEM.md`, `useDensity`).

## Архитектура (обязательное решение)
- **Публичный API:** один компонент `CardListColumn` (и существующие / расширенные типы), экспорт из `packages/ui/src/index.ts` без нового обязательного импорта «второго» компонента для того же сценария.
- **Внутри пакета:** разнести разметку/стили трёх видов по вспомогательным модулям (например `CardListColumnCollapsed.tsx`, `CardListColumnListBody.tsx`, `CardListColumnGridBody.tsx`) или эквивалент; общее состояние списка (фильтр, сортировка, скролл, load more) не дублировать.
- **Состояние вида:** тип вида `collapsed` | `list` | `grid` (имена зафиксировать в коде и в этой задаче). Переключение «по кругу» = переход к следующему виду с зацикливанием.
- **Контролируемость (рекомендуется):** опциональные пропсы `view` + `defaultView` + `onViewChange`, чтобы продукт мог синхронизировать вид с URL или сохранять в `localStorage`; при отсутствии — только внутреннее состояние.

## Входит в объем

### 1. Данные и выбор
- Расширить **`CardListColumnItem`** (обратно совместимо): например опциональные `imageUrl?: string` и/или `avatarIcon?: AprilLucideIcon` (или согласованный тип из `lucide-react` — как принято в пакете для пропов «иконка»).
- Добавить поддержку **выбранной карточки**: как минимум `selectedItemId?: string | null` и `onSelectItem?: (id: string | null) => void` (или эквивалентная пара controlled/uncontrolled с `defaultSelectedItemId`). Поведение клика по строке/карточке в списке и в сетке — выбор; повторный клик по активной — уточнить в PR (обычно оставить выбранной или снять выбор; зафиксировать одно правило в коде и витрине).

### 2. Вид «свернуто» (`collapsed`)
- Узкая колонка (текущий порядок величин ~72px или осмысленно пересмотреть при двух строках текста — не ломая Kanban-rail).
- Показывать **название выбранной** карточки (по `selectedItemId` → `title`); при отсутствии выбора — нейтральная заглушка (короткая строка, согласованная с витриной).
- Длинные названия: **обрезка** (`truncate` / `lineClamp`) + полный текст в **`title`** нативного элемента и/или `Tooltip` при hover/focus (a11y §10).
- Кнопка цикла видов: доступное имя (`aria-label`) отражает **следующий** вид или **текущий** + тултип — выбрать один стиль и применить везде; иконки — `AprilIcon*` / Lucide по `DESIGN_SYSTEM.md` §11.

### 3. Вид «список» (`list`)
- Текущая колонка с панелью управления и вертикальным списком; **дефолтная** строка карточки (если не задан `renderCard`): слева **`Avatar`** Mantine с `src={imageUrl}`; при отсутствии URL — **иконка**-fallback (из item или дефолтная одна на весь список через проп, например `defaultListItemIcon`).
- Визуально выделить **selected** (фон/бордер `theme`) — согласованно с темой light/dark и плотностью.

### 4. Вид «сетка» (`grid`)
- Список карточек в **CSS Grid**: заполнение **слева направо, сверху вниз**, карточки **занимают доступную высоту и ширину** области контента колонки (в смысле «весь main колонки», без искусственного ограничения по высоте списка, если `heightMode="fill"`).
- Карточка в стиле, вдохновлённом [Tasks card](https://ui.mantine.dev/component/task-card/): заголовок, опционально вторичный текст/progress-плейсхолдер при необходимости для витрины, аватар/иконка; без копирования чужих цветов.
- Сохранить **load more** при скролле, если он уже есть для списка (адаптировать под viewport сетки).
- Ресайз правой границы: уточнить в PR — в режиме `grid` ширина колонки **как в list** или фиксирована; зафиксировать поведение в `TASK`/коде и витрине.

### 5. Режимы `inline` / `overlay`
- Существующее поведение не ломать; три вида должны работать в обоих режимах там, где это осмысленно (overlay — поверх контента, как сейчас).

### 6. Документация и витрина
- **`DESIGN_SYSTEM.md` §13** — обновить описание Card List Column: три вида, выбор, аватар, ссылка на референс сетки.
- **`UIKit` / showcase:** секция с `CardListColumn` — демо переключения видов, выбранный элемент, элементы с картинкой и без; comfortable/compact.
- Обновить **`CardListColumn.smoke.test.tsx`** (и при необходимости секцию витрины/селекторы) под новые элементы и `aria-label`.

### 7. Проверки
- `pnpm lint`, `pnpm typecheck`, `pnpm build`, `pnpm test` (как в CI).

## Не входит в объем
- Виртуализация длинных списков и сеток (можно завести follow-up, если замер покажет проблему).
- Drag-and-drop между колонками.
- Отдельный публичный компонент только для сетки (противоречит цели «один импорт»).
- Загрузка изображений с внешних доменов, CDN-оптимизация, `next/image` — зона продукта.

## Ограничения
- Монорепозиторий на pnpm; не импортировать `UIKit` в production-код пакета `ui`.
- Иконки UI — `AprilIcon*` / `lucide-react` по §11; не вводить второй набор для того же смысла.
- Модалки фильтра/сортировки/add — по-прежнему **`AprilModal`** и паттерн шапки DS-010.
- Не хардкодить произвольные hex; плотность — через `useDensity` там, где меняются размеры контролов и отступов.
